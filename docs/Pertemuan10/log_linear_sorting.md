# Log Linear Sorting

Pada praktikum sebelumnya, kita mempelajari algoritma sorting dasar seperti Insertion Sort, Selection Sort, dan Bubble Sort. Algoritma ini **mudah dipahami, tetapi kurang efisien untuk data berukuran besar** karena memiliki kompleksitas waktu **(O(n²))**. Ini berarti **semakin banyak data, semakin lama proses pengurutannya.**

Untuk mengatasi hal tersebut, digunakan algoritma yang lebih optimal seperti Merge Sort dan Quick Sort. Algoritma ini termasuk **Log-Linear Sorting Algorithms**, yaitu kelompok algoritma sorting yang memiliki kompleksitas waktu rata-rata **O(nlogn)**.

Algoritma log-linear bekerja lebih efisien karena menggunakan strategi **Divide and Conquer**, yaitu **membagi masalah besar menjadi bagian-bagian kecil, lalu menyelesaikannya secara bertahap hingga data terurut seluruhnya.**

Dibandingkan algoritma quadratic sorting, algoritma log-linear mampu menangani data berukuran besar dengan performa yang jauh lebih **cepat dan stabil.** Oleh karena itu, algoritma seperti Merge Sort dan Quick Sort banyak digunakan pada database, sistem operasi, mesin pencarian, dan berbagai aplikasi modern lainnya.
